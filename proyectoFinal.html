Objetivo General

Crear un sistema de Bases de Datos Relacionales Normalizado que permita que el usuario o administrador pueda tener acceso y realizar consultas, crear nuevos registros y eliminar información a través de un sistema Web.

Objetivos Particulares

  • Diseñar un sistema de base de datos relacional en la tercera forma normal.

  • Implementar el diseño de la base de datos en el gestor MySQL/MariaDB.

  • Crear un sistema web HTML que permita la consulta, inserción, modificación y búsqueda de información.

  • Utilizar CSS para el manejo de estilo del sistema web y Bootstrap para que el sitio sea responsivo.

  • Utilizar PHP para el manejo de la interacción del sistema web con la base de datos.

Requerimientos

  1. Se deberá elegir una temática particular para la información que se maneja en el sistema.

  2. Diseñar la base de datos relacional que contenga al menos 3 relaciones y 8 tablas en total.

    1. Realizar la normalización a la tercera forma normal (3FN).

      1. Para realizar la normalización deben definir con claridad sus relaciones, las entidades en cada una de ellas y las dependencias funcionales.

      2. Mientras más dependencias funcionales encuentren, menor redundancia habrá en la base de datos.

      3. El algoritmo de normalización entrega las tablas ya normalizadas de la relación en cuestión.

    2. Realizar los diagramas de la base de datos relacional.

  3. Implementar la base de datos en el gestor de base de datos MySQL creando las entidades, tablas y llaves correspondientes.

    1. Llenar la base de datos con algunos datos de prueba, de manera que se puedan realizar búsquedas posteriormente.

  4. Crear un sistema web con HTML5 + CSS3 + Bootstrap

    1. Utilizar elementos de HTML5.

    2. Hacer un estilo básico del sistema a través de CSS.

    3. Utilizar Bootstrap para que el sitio sea responsivo (en particular en tablas, formularios e imágenes).

  5. Utilizar PHP para la interacción del sistema web con la base de datos.

    1. El sistema debe permitir inserción de registros, debe permitir seleccionar la entidad y tabla en la que se hará dicha inserción. La inserción se debe hacer a través de un formulario de HTML.

    2. La consulta de información debe hacerse solicitando al usuario la entidad y tabla que desea mostrarse. Luego mostrar el contenido de la tabla seleccionada en una tabla de HTML.

    3. La eliminación de la información se deberá hacer a través de un formulario de HTML, siendo necesario solamente el valor de la llave primaria de la tupla a borrar.

  6. Para fines del proyecto usar el programa XAMPP o alguna otra alternativa.

  7. Seguir la estructura de archivos siguientes en el servidor, creando las carpetas y el archivo html correspondiente:

    1. css: contiene todos los archivos de estilo CSS

    2. img: contiene todos los archivos de imágenes en el sitio (si utilizan alguna)

    3. index.html o index.php: el archivo inicial del sistema. Este archivo se encuentra en el llamado "directorio raíz del sistema", en esta ubucación deberá colocar todos los archivos html y/o php que escriba para su sistema.

Entrega

La entrega del proyecto consiste en un documento que contenga lo siguiente:

  1. Introducción: Una breve descripción de lo que es y hace el sistema

  2. Diseño: Los diagramas del diseño de la base de datos relacional. Esta es la parte central del proyecto, ya que es uno de los principales objetivos de la materia.

  3. Normalización: El procedimiento de normalización de la base de datos a 3FN.

  4. Conclusiones: Unas breves palabras acerca de lo que aprendieron al realizar el proyecto final y todo el curso en sí.
