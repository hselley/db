Introducción

SQL es el DBMS (DataBase Management System) más comúnmente utilizado para consultar y modificar una base de datos relacional a través de un lenguaje llamado SQL (Structured Query Language (Lenguaje Secuencial de Consultas)). Una de sus características es el manejo del álgebra y el cálculo relacional que permiten efectuar consultas con el fin de recuperar de forma sencilla información de interés de una base de datos, así como hacer cambios en ellas.

SQL:

  • SQL es el acrónimo de Structured Query Language (Lenguaje Secuencial de Consultas).

  • SQL permite tener acceso y manipular bases de datos.

  • SQL es un estándar ANSI (American National Standards Institute (Instituto de Estandres Nacionales Americanos))

SQL es capaz de:

  • Realizar consultas en una base de datos.

  • Obtener información de una base de datos.

  • Insertar registros en una base de datos.

  • Borrar registros en una base de datos.

  • Crear nuevas bases de datos.

  • Crear nuevas tablas en una base de datos.

  • Crear procedimientos de almacenamiento en una base de datos.

  • Crear vistas en una base de datos.

  • Asignar permisos en las tablas, procedimientos y vistas.

Historia

Los orígenes de SQL están ligados a los de las bases de datos relacionales.

En 1970 F. Codd propone el modelo relacional y asociado a este un sublenguaje de acceso a base de datos basado en el cálculo de predicados.

Basándose en esas ideas, los laboratorios de IBM definieron el lenguaje SEQUEL (Structured English Query Language) que más tarde fue ampliamente implementado por el sistema de gestión de bases de datos relacionales surgidos en los años siguientes y fue por fin estandarizado en 1986 por el ANSI, dando lugar a la primera versión estándar de este lenguaje, el "SQL-86" o "SQL1". Al año siguiente este estándar es también adoptado por la ISO.

Características

El SQL es un lenguaje de acceso a bases de datos que explota la flexibilidad y potencia de los sistemas relacionales y permite así gran variedad de operaciones.

Es un lenguaje declarativo de "alto nivel" o "de no procedimiento" que, gracias a su fuerte base teórica y su orientación al manejo de conjuntos de registros —y no a registros individuales— permite una alta productividad en codificación y la orientación a objetos.

SQL tiene las siguientes características:

  • Lenguaje de definición de datos: El LDD de SQL proporciona comandos para la definición de esquemas de relación, borrado de relaciones y modificaciones de los esquemas de relación.

  • Lenguaje interactivo de manipulación de datos: El LMD de SQL incluye lenguajes de consultas basado tanto en álgebra relacional como en cálculo relacional de tuplas.

  • Integridad: El LDD de SQL incluye comandos para especificar las restricciones de integridad que deben cumplir los datos almacenados en la base de datos.

  • Definición de vistas: El LDD incluye comandos para definir las vistas.

  • Control de transacciones: SQL tiene comandos para especificar el comienzo y el final de una transacción.

  • SQL incorporado y dinámico: Esto quiere decir que se pueden incorporar instrucciones de SQL en lenguajes de programación como: C++, C, Java, PHP, Cobol, Pascal y Fortran.

  • Autorización: El LDD incluye comandos para especificar los derechos de acceso a las relaciones y a las vistas.

A pesar de que SQL es un estándar ANSI (American National Standards Institute), existen diferentes versiones del lenguaje.

Para poder utilizar SQL en un sitio web, de manera que el sitio muestra información desde la base de datos, se necesita:

  • Un RDBMS (Sistema de Gestión de Bases de Datos Relacionales), por ejemplo: MS Access, MS SQL Server, MySQL, IBM DB2, Oracle SQL, etc.

  • Utilizar un lenguaje script del lado del servidor, por ejemplo: PHP o ASP.

  • Utilizar SQL para obtener la información que requiere.

  • Utilizar HTML / CSS.

Sentencias SQL

Muchas de las acciones que se realizan en una base de datos se hacen mediante sentencias SQL. El siguiente ejemplo selecciona todos los registros de la tabla Customers:

Ejemplo 1. Sentencia Select 


      select * from Customers;

Tenga en mente que SQL no es sensible a las mayúsculas: select significa lo mismo que SELECT o inclusive SeLeCt.

Algunos sistemas de bases de datos requieren un punto y coma al final de cada sentencia. El punto y coma es una forma estándar de separar sentencias SQL en sistemas de base de datos que permiten más de una sola sentencia en una sola llamada al servidor.

Algunos de los comandos más importantes en SQL

  • SELECT: Extrae información de la base de datos.

  • UPDATE: Actualiza información de la base de datos.

  • DELETE: Borra información de la base de datos.

  • INSERT INTO: Insertar nueva información en la base de datos.

  • CREATE DATABASE: Crea una nueva base de datos.

  • ALTER DATABASE: Modifica una base de datos.

  • CREATE TABLE: Crea una nueva tabla en la base de datos.

  • ALTER TABLE: Modifica una tabla.

  • DROP TABLE: Borra una tabla.

  • CREATE INDEX: Crea un índice (llave de búsqueda).

  • DROP INDEX: Borra un índice.

Sentencia Select

La sentencia SELECT se utiliza para seleccionar información en una base de datos. El resultado se almacena en una tabla, llamada "conjunto resultado".

Sintaxis Select


      SELECT nombre_columna, nombre_columna FROM nombre_tabla;

y 


      SELECT * FROM nombre_tabla;

Ejemplo 2. La siguiente sentencia SQL selecciona los atributos CustomerName y City de la tabla Customers. 


      SELECT CustomerName, City FROM Customers;

Ejemplo 3. La siguiente sentencia SQL selecciona todos los atributos de la tabla Customers. 


      SELECT * FROM Customers;

Muchos de los sistemas de bases de datos permiten la navegación en el conjunto resultado con funciones programadas, como: ir al primer registro, obtener contenido del registro, mover al siguiente registro, etc.

Sentencia SELECT DISTINCT

En una tabla, una columna puede contener muchos valores duplicados y algunas veces puede necesitar solamente una lista de valores diferentes. La palabra clave DISTINCT puede ser utilizada para devolver solamente valores diferentes.

Sintaxis


      SELECT DISTINCT column_name,column_name FROM table_name;

Ejemplo 4. La sentencia SQL solo selecciona los valores diferentes de la columna City de la tabla Customers. 


      SELECT DISTINCT City FROM Customers;

Sentencia WHERE

La sentencia WHERE se utiliza para filtrar registros. Esta cláusula sólo devuelve aquellos registros que cumplen cierta condición especificada.

Sintaxis


      SELECT column_name,column_name
      FROM table_name
      WHERE column_name operator value;

Ejemplo 5. La sentencia SQL selecciona todos los clientes cuyo país sea México en la tabla Customers. 


      SELECT * FROM Customers WHERE Country='Mexico';

Campos de Texto vs. Campos Numéricos

SQL requiere de comillas sencillas alrededor de valores tipo texto (muchos sistemas de bases de datos también permiten comillas dobles). Sin embargo, los valores numéricos no deben ir entre comillas.

Ejemplo 6. 


      SELECT * FROM Customers WHERE CustomerID=1;

Operador Descripción

Operadores AND y OR

Los operadores AND y OR son utilizados para filtrar registros basados en más de una condición.

El operador AND despliega un registro si las dos condiciones son verdaderas.

El operador OR despliega un registro si alguna de las condiciones es verdadera.

Operador AND

Ejemplo 7. La sentencia SQL selecciona todos los clientes cuyo país sea Germany y su ciudad sea Berlín en la tabla Customers. 


      SELECT * FROM Customers WHERE Country='Germany' AND City='Berlin';

Operador OR

Ejemplo 8. La sentencia SQL selecciona todos los clientes cuya ciudad sea Berlín o München en la tabla Customers. 


      SELECT * FROM Customers WHERE City='Berlin' OR City='München';

Combinando AND y OR

Es posible también combinar AND y OR para hacer un filtro específico de un conjunto de registros.

Ejemplo 9. La sentencia SQL selecciona todos los clientes cuyo país sea Germany y la ciudad sea igual a Berlin o München en la tabla Customers. 


      SELECT * FROM Customers WHERE Country='Germany' AND (City='Berlin' OR City='München');

Palabra clave ORDER BY

La palabra clave ORDER BY sirve para ordenar una o más columnas del conjunto resultado. Ésta palabra ordena los registros en forma ascendente por defecto, para ordenarlas en orden descendente se usa la palabra DESC.

Sintaxis


      SELECT column_name, column_name FROM table_name ORDER BY column_name ASC|DESC, column_name ASC|DESC;

Ejemplo 10. La sentencia SQL selecciona todos los clientes de la tabla Customers y los ordena por la columna Country. 


      SELECT * FROM Customers ORDER BY Country;

Pruebe las siguientes consultas y explique su funcionamiento: 


      SELECT * FROM Customers ORDER BY Country DESC;
      

      SELECT * FROM Customers ORDER BY Country, CustomerName;
      

      SELECT * FROM Customers ORDER BY Country ASC, CustomerName DESC;

Sentencia INSERT INTO

La sentencia INSERT INTO se utiliza para insertar registros nuevos en una tabla.

Sintaxis

Es posible utilizar la sentencia INSERT INTO de dos formas diferentes.

En la primera forma no se especifican los nombres de las columnas donde los datos serán insertados, sólo sus valores: 


      INSERT INTO table_name VALUES (value1,value2,value3,...);

La segunda forma se especifica el nombre de la columna y los valores que serán insertados. 


      INSERT INTO table_name (column1,column2,column3,...) VALUES (value1,value2,value3,...);

Ejemplo 11. La sentencia SQL añade un nuevo registro a la tabla Customers. 


      INSERT INTO Customers (CustomerName, ContactName, Address, City, PostalCode, Country)
        VALUES ('Cardinal','Tom B. Erichsen','Skagen 21','Stavanger','4006','Norway');

También es posible insertar datos en columnas específicas.

La siguiente sentencia inserta un nuevo registro pero solamente inserta información en las columnas CustomerName, City y Country. (el campo CustomerID se llena automáticamente).

Ejemplo 12. 


      INSERT INTO Customers (CustomerName, City, Country)
      VALUES ('Cardinal', 'Stavanger', 'Norway');

Sentencia UPDATE

La sentencia UPDATE se utiliza para actualizar registros en una tabla.

Sintaxis


      UPDATE table_name
      SET column1=value1,column2=value2,...
      WHERE some_column=some_value;

Notese que la cláusula WHERE especifica que registro(s) será(n) actualizado(s), si se omite, todos los registros serán actualizados.

Ejemplo 13. La sentencia SQL actualiza información de un cliente. 


      UPDATE Customers
      SET ContactName='Alfred Schmidt', City='Hamburg'
      WHERE CustomerName='Alfreds Futterkiste';

Ejemplo 14. Omisión de WHERE 


      UPDATE Customers
      SET ContactName='Alfred Schmidt', City='Hamburg';

Sentencia DELETE

La sentencia DELETE se utiliza para borrar registros en una tabla.

Sintaxis


      DELETE FROM table_name
      WHERE some_column=some_value;

Ejemplo 14. La sentencia SQL borra al cliente Alfreds Futterkiste de la tabla Customers. 


      DELETE FROM Customers
      WHERE CustomerName='Alfreds Futterkiste' AND ContactName='Maria Anders';

Borrar todos los datos

Es posible borrar todos los renglones en una tabla sin borrar la tabla. Esto significa que la estructura de la tabla, atributos e índices quedarán intactos. 


      DELETE FROM table_name;

o 


      DELETE * FROM table_name;

Sentencia CREATE DATABASE

La sentencia CREATE DATABASE se utiliza para crear una base de datos.

Sintaxis


      CREATE DATABASE dbname;

Ejemplo 15. La sentencia SQL crea una base de datos llamada my_db. 


      CREATE DATABASE my_db;

Sentencia CREATE TABLE

La sentencia CREATE TABLE se utiliza para crear tablas en una base datos. Las tablas están organizadas en renglones y columnas y cada una debe tener nombre.

Sintaxis


      CREATE TABLE table_name (
        column_name1 data_type(size),
        column_name2 data_type(size),
        column_name3 data_type(size),
        ...
      );

El parámetro column_name especifica el nombre de la columna. El parámetro data_type especifica que tipo de dato almacena la columna (varchar, integer, decimal, date, etc.). El parámetro size especifica el tamaño máximo de la columna.

Ejemplo 16. Crear una tabla llamada Persons que contiene 5 columnas: PersonID, LastName, FirstName, Address y City. 


      CREATE TABLE Persons (
        PersonID int,
        LastName varchar(255),
        FirstName varchar(255),
        Address varchar(255),
        City varchar(255)
      );

Restricción PRIMARY KEY

La restricción PRIMARY KEY identifica de forma única cada registro en la tabla de la base de datos. La llave primaria debe contener valores únicos en la tabla. Una columna de llave primaria no puede contener valores nulos, por ello se agrega el término NOT NULL. Se añade AUTO_INCREMENT para que el atributo tome un valor incremental en 1 automáticamente cada vez que se añade un registro a la tabla. La mayoría de las tablas deberán tener una llave primaria, y cada tabla puede tener solo una llave primaria.

Sintaxis MySQL


      CREATE TABLE Persons (
        P_Id int NOT NULL AUTO_INCREMENT,
        LastName varchar(255) NOT NULL,
        FirstName varchar(255),
        Address varchar(255),
        City varchar(255),
        PRIMARY KEY (P_Id)
      )

PRIMARY KEY en ALTER TABLE


      ALTER TABLE Persons
      ADD PRIMARY KEY (P_Id);

DROP una PRIMARY KEY


      ALTER TABLE Persons
      DROP PRIMARY KEY;
